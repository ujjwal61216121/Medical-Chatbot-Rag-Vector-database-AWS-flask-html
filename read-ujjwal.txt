Project Explanation

This project is a RAG-based Medical Chatbot built using Flask, LangChain, Pinecone, Groq, and HuggingFace embeddings.

We used a medical book PDF dataset as the knowledge base for the chatbot.

The workflow of the project is:

The medical PDF book is loaded and split into smaller text chunks.
These chunks are converted into vector embeddings using the embedding model:
sentence-transformers/all-MiniLM-L6-v2
The generated embeddings are stored inside Pinecone, which acts as the vector database.
When a user asks a question, Pinecone performs semantic similarity search and retrieves the most relevant chunks from the stored medical data.
The retrieved context is then passed to the Groq Llama 3.1 model through the Groq API to generate the final response.
The backend logic is implemented using Flask, while the frontend UI is developed using HTML, CSS, Bootstrap, and JavaScript.
